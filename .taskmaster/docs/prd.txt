# 개요
이 프로젝트는 개인 포트폴리오 웹사이트입니다. 주요 목표는 본인의 경력, 프로젝트, 기술 스택, 글(블로그/노트)을 명확하고 인상 깊게 전달하여 채용/협업 기회를 확대하는 것입니다. 1) 빠른 로딩과 반응형 UI, 2) 검색/필터 가능한 프로젝트/글 목록, 3) 손쉬운 콘텐츠 관리가 핵심 가치입니다.

# 핵심 기능
- 프로젝트 소개
  - 수행한 프로젝트 카드 리스트(썸네일, 제목, 한 줄 요약, 기술스택 태그, 기간, 링크)
  - 상세 페이지(문제/역할/성과/기술/스크린샷/데모 링크)
  - 태그/텍스트 검색 및 필터
- 소개(About)
  - 한 줄 소개, 상세 이력 요약, 다운로드 가능한 이력서 링크
  - 연락처/소셜 링크(Email, GitHub, LinkedIn 등)
- 블로그/노트(선택)
  - 글 목록(카테고리/태그/검색)
  - 글 상세(Markdown 렌더링, 코드 하이라이트)
- 연락(Contact)
  - 폼 제출(스팸 방지, 성공/실패 알림)
  - 메일 전송 또는 외부 폼 서비스 연동
- 다국어(선택): ko/en 토글
- SEO/공유 최적화: 메타 태그, OG/Twitter 카드, 사이트맵, RSS(선택)
- 성능 최적화: 이미지 최적화(LQIP/blur), 코드 스플리팅, 정적 빌드 우선
- 접근성: 색 대비, 키보드 탐색, aria 속성 점검

# 사용자 경험
- 페르소나
  - 리크루터/매니저: 빠르게 요약/성과/대표작 확인
  - 동료 개발자/디자이너: 구현/코드/디자인 디테일 확인
  - 일반 방문자: 프로젝트 스토리와 데모 위주로 소비
- 핵심 플로우
  - 홈 → 대표 프로젝트 하이라이트 → 전체 프로젝트 목록 → 상세 페이지
  - 홈/소개 → 이력서 다운로드 → 연락
  - 홈/블로그 → 글 상세 → 관련 글/프로젝트 유도
- UI/UX 고려사항
  - 깔끔한 다크/라이트 테마(선택)
  - 그리드 카드 레이아웃, 부드러운 트랜지션
  - 모바일 퍼스트, 360px~ 데스크톱까지 반응형

# 기술 아키텍처
- 프레임워크: 정적 사이트 생성(SSG) 우선. Next.js, Astro, SvelteKit 등 중 현재 레포 상태에 맞춰 선택(구현 단계에서 확정).
- 데이터 모델
  - Project: id, title, summary, description(md), techTags[], period, thumbnail, links(demo, repo), featured(boolean)
  - Post(선택): id, title, slug, content(md), tags[], date, cover
  - Profile: name, title, bio, skills[], socials, resumeUrl
- 콘텐츠 관리
  - 최소 구현: Markdown 기반 로컬 콘텐츠 관리
  - 선택: Notion/CMS 연동 가능성을 염두(추후 확장)
- 통합/인프라
  - 배포: Vercel/Netlify/GitHub Pages 중 하나(레포 환경에 맞춤)
  - CI: PR 미리보기, 품질 체크(lint/typecheck/build)
- 품질
  - TypeScript(가능 시), ESLint, Prettier, 이미지 최적화, Lighthouse 90+ 목표

# 개발 로드맵
- MVP
  - 홈, 소개, 프로젝트 목록/상세, 기본 SEO, 성능 최적화, 접근성 기초
  - Markdown 로컬 콘텐츠, 간단한 연락 폼(외부 서비스 연동 또는 mailto 대안)
- 확장
  - 블로그 섹션, 태그/검색 고도화, 다국어 지원, RSS/사이트맵, 애널리틱스
  - CMS/Notion 연동, 다크 모드, 애니메이션 디테일

# 논리적 의존성 체인
- 디자인 시스템/레이아웃 기초 → 데이터 스키마/콘텐츠 구조 → 목록/상세 페이지 → SEO/성능/접근성 → 배포/분석 → 확장 기능(블로그/다국어/CMS)

# 리스크 및 대응
- 콘텐츠 부족: 템플릿/더미 데이터로 초기 구성 후 점진 교체
- 일정/리소스: MVP 범위를 엄격히 관리, 이후 점진 확장
- 기술 선택: 현재 레포/호스팅 환경에 맞춰 의사결정, 복잡도 최소화
